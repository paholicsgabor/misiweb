1. Git branch-ek és verziókezelés

Ma megtanultuk, hogyan lehet több branch-et (ágat) használni a Git-ben:

  • main – ez a fő branch, ami a szerveren fut ("éles verzió")
  • local-updates – ide mentettük a helyi változtatásokat
Fontos parancsok:
git checkout main – átváltás a main branch-re
git checkout local-updates – átváltás a helyi branch-re
git push origin local-updates – feltöltés GitHub-ra

2. Git stash – változtatások félretétele

A git stash paranccsal félre tudjuk tenni a fájlokat anélkül, hogy commitolnánk:

  • Kivehetjük az mp3 fájlokat, amiket nem akarunk a repóban
  • Később visszahozhatjuk: git stash pop
Amit félretettünk: mp3 fájlok, ChatGPT mappa, memoria.py, index.html, make-tts.js

3. Webhook – automatikus deploy

Létrehoztunk egy webhook rendszert. Ez azt jelenti:

Te push-olsz a GitHub-ra (main branch) | v GitHub küld egy üzenetet a szerverednek | v A szerver automatikusan letölti az új kódot | v PM2 újraindítja az alkalmazást | v Az oldal frissül magától!
  • A webhook listener egy kis Node.js program a szerveren (port 9000)
  • Titkos kulccsal védjük, hogy ne csinálhasson bárki deploy-t
  • A PM2 kezeli – automatikusan fut a háttérben

4. HTTPS és Let's Encrypt

Az oldalad most már HTTPS-en fut, ami azt jelenti, hogy titkosított a kapcsolat:

  • HTTP = olvasható (mint egy képeslap – bárki elolvassa)
  • HTTPS = titkosított (mint egy leragasztott levél)
Amit beállítottunk:
- Nginx – reverse proxy (kívülről jövő kéréseket továbbítja a Node.js-nek)
- Let's Encrypt – ingyenes HTTPS tanúsítvány
- Certbot – automatikusan megújítja a tanúsítványt
- A domain: https://misiweb.it

5. Nginx – reverse proxy

Az Nginx egy "kapuőr" a szerver előtt:

Felhasználó böngészője | v misiweb.it:443 (HTTPS) | v NGINX (kapuőr) / \ / \ v v port 3000 port 9000 (weboldal) (webhook)
  • A felhasználó a 443-as portra csatlakozik (HTTPS)
  • Az Nginx továbbítja a kérést a Node.js-nek (port 3000)
  • A /webhook útvonalat a webhook listener-nek (port 9000)

6. AWS Security Group

Az AWS Security Group olyan, mint egy tűzfal – megmondja, melyik porton enged be forgalmat:

Szükséges portok:
- Port 22 – SSH (szerver kezelés)
- Port 80 – HTTP (átirányítás HTTPS-re)
- Port 443 – HTTPS (az oldal maga)

7. Amit még be kell fejezni

  • AWS Security Group-ban megnyitni a 80 és 443 portokat
  • GitHub Webhook beállítása a repo settings-ben:
    • URL: https://misiweb.it/webhook
    • Secret: mnp-webhook-secret-2026
  • Tesztelni, hogy a push után az oldal frissül-e
